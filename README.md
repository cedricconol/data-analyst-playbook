## Start Here

- Beginners with zero experience  
- Career shifters

### Who this is NOT for ❌
- Experienced analysts

### How to use this playbook

Follow the phases in the order listed here. Try to focus on execution, not perfection.
> [Fail fast, fail forward](https://medium.com/@vijaisenthil/the-psychology-behind-fail-fast-fail-forward-c90b92b20f76), learn faster!


Learning objectives are listed for each phase with links to free resources to help you achieve the phase objectives. You may choose to use other sources as you feel necessary.

## What Data Analysts Actually Do

"Data Analysts answers questions and solves problems using data."

1. Start by defining the business problem or question
2. Collect the data required for the analysis
3. Explore and understand the data
4. Prepare the data for analysis: clean, model, and validate
5. Analyze to derive conclusions and answers to the problem or question at hand
6. Present and share the results of the analysis

## The Curriculum


### Phase 1 — Thinking Like an Analyst

**Objectives:**
- Understanding business questions  
- Breaking problems into smaller parts  
- Asking better questions

**Resources:** 
- [Step-by-Step Process of Thinking Like a Data Analyst](https://codebasics.io/blog/step-by-step-process-of-thinking-like-a-data-analyst)

---

### Phase 2 — Excel / Google Sheets

**Objectives:**
- Understand lookup functions (vlookup, xlookup)
- Learn pivot tables
- Visualization in spreadsheets
  
**Resources:**
- [Microsoft Excel Tutorial for Beginners - Full Course](https://www.youtube.com/watch?v=Vl0H-qTclOg) - you don't have to watch the entire video, feel free to stop when learning objectives are met

**Practice:**  
- Use any dataset of interest from [Kaggle Datasets](https://www.kaggle.com/datasets) 
- Build 1 simple report using pivot tables and charts

---

### Phase 3 — SQL

**Objectives:**
- Master the basic syntax: SELECT, FROM, WHERE, GROUP BY, and ORDER BY
- Master control flow functions: CASE, IF, IFNULL, NULLIF, and COALESCE
- Understand the different types of JOINs
- Learn SQL order of execution
- Bonus: Learn window functions such as ROW_NUMBER, RANK, LEAD, LAG

**Resource:**  
- [SQLBolt](https://sqlbolt.com/) 


**Practice:**
- [LeetCode: SQL 50](https://leetcode.com/studyplan/top-sql-50/)
- Do 10–20 easy and 5 medium problems

---


### Phase 4 — Dashboards & Storytelling

**Objectives:**
- Different chart types and when to use each
- Storytelling with dashboards

**Resources:**
- [Power BI Tutorial From Beginner to Pro ⚡ Desktop to Dashboard in 60 Minutes ⏰](https://www.youtube.com/watch?v=AGrl-H87pRU)

**Practice:**
- Follow along the YouTube course and build a dashboard, use the same dataset you used in Phase 2 

---

### Phase 5 — Python (Optional)

**Objectives:**
- Understand the basics such as data types, control flow, functions
- Data analysis using Pandas

**Resources:**  
- [Complete Python Pandas Data Science Tutorial! (Reading CSV/Excel files, Sorting, Filtering, Groupby)](https://www.youtube.com/watch?v=vmEHCJofslg)

**Practice:**
- Perform SQL-based analysis but in Python Pandas

---

### Phase 6 — Git

**Objectives:**
- What is version control and why is it important
- Basic commands such as clone, pull, add, commit, push, rebase
- How the Pull Request process works

**Resources:**
- [Git and GitHub for Beginners - Crash Course](https://www.youtube.com/watch?v=RGOj5yH7evk)


**Practice:**
- Create a GitHub account and create a repository
- Upload your SQL scripts to the repository
- Make modifications and push changes to the repository

---

### Phase 7 — AI for Data Analysts

**Objectives:**
- Familiarize ChatGPT and Claude
- Learn how to write prompts
- Use coding assistants such as Cursor and Copilot

**Resources:**
- [Cursor Docs](https://cursor.com/docs/get-started/quickstart)

**Practice:**
- Ask AI to explain a code snippet
- Ask AI to write code for you
- Ask AI to check your code 

⚠️ Don’t blindly copy. Always understand the output.

## Putting Everything Together

### Passion Project
1. Find a dataset about a topic you are highly interested: stock prices, NBA data etc.
2. Formulate a business question that you want answered
3. Create a Github repository and name it after your project
4. Clone the repository locally and use Cursor as your IDE
5. Setup MySQL + PowerBI following [this guide](https://www.youtube.com/watch?v=tWcGaNmvHk0). For Mac users, use Parallels to make this work.
6. All SQL code used to generate the data for PowerBI charts must be synced and version controlled in Github.
7. Write a Summary in the repository README, add screenshots of your charts to tell the story. You may need to learn [Markdown syntax](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/quickstart-for-writing-on-github).
